# 网页版云注入平台 - 产品需求文档（PRD）

## 1. 产品概述

基于 `laobi465/wlyz`（KeyAuth SaaS 多租户卡密验证平台）的核心安全设计理念，构建一个**专注云注入（Cloud Injection）场景的高安全性 Web 平台**。开发者通过 Web 后台管理云变量/注入脚本，客户端软件通过卡密 + HMAC 签名 + 硬件指纹三重验证后从云端拉取注入内容；所有敏感数据全程加密，所有操作可审计。

- **目标用户**：需要远程动态下发配置/脚本/变量到客户端软件的开发者（独立开发者、小型团队、安全研究场景）
- **核心价值**：在 Web 上完成"卡密签发 → 硬件绑定 → 云端注入内容下发 → 实时审计"的完整闭环，安全等级对标原项目 v0.6.6 的 P0+P1 修复后状态

## 2. 核心功能

| 角色 | 注册方式 | 核心权限 |
|------|---------|---------|
| 平台超管（admin） | 首次初始化 | 全平台管理 + 安全配置 + 审计查看 |

### 2.2 功能模块

1. **登录页**：账号密码 + TOTP 2FA 双因子 + 暴力破解自动锁定
2. **工作台 Dashboard**：应用数/卡密数/今日验证次数/在线设备数 + 24h 趋势图
3. **应用管理**：创建应用、生成 AppKey/AppSecret/SignSecret（AES 加密入库）、密钥轮换
4. **卡密管理**：5 类型卡密（duration/count/permanent/trial/feature）、批量生成、状态机、硬件指纹绑定查询
5. **云注入变量**（核心）：CRUD 云变量、值类型（string/number/json/bool）、AES-256-GCM 加密存储、只读标记、按应用过滤
6. **设备/会话**：在线设备列表（Redis Sorted Set 模拟）、硬件指纹详情、强制下线
7. **审计日志**：登录日志、操作日志、验证日志三表分离、CSV 导出
8. **安全中心**：2FA 启用/备用码、IP 黑白名单、登录设备管理、修改密码

## 3. 核心流程

**开发者下发云注入流程**：开发者登录后台 → 选择应用 → 创建云变量（值 AES-256-GCM 加密入库）→ 客户端软件启动 → 卡密验证 + 硬件指纹绑定 + HMAC-SHA256 签名 → 调用 `/api/v1/client/get_var` → 服务端校验卡密有效+签名+nonce 防重放 → 解密云变量返回 → 客户端注入到运行时。

**安全验证流程（每次客户端请求）**：
1. 校验 IP 黑名单（Redis 故障回退 SQLite）
2. 校验 AppKey 存在 + 签名 `HMAC-SHA256(SignSecret, method+path+timestamp+nonce+body)`
3. 校验 timestamp 偏差 ≤ 5 分钟
4. 校验 nonce 未被使用过（写入 Redis/内存 5 分钟 TTL）
5. 校验卡密有效（状态 + 到期 + 硬件指纹匹配）
6. 写入审计日志（异步）

## 5. 安全性要求（重点）

### 5.1 认证与授权

- 密码 bcrypt cost=12，禁止弱密码（<8 位 / 纯数字 / 常见密码字典）
- JWT access token 15 分钟 + refresh token 7 天轮换 + jti 黑名单
- JWT 强制校验 `Subject == "access"`，refresh 不能访问业务接口
- TOTP 2FA（RFC 6238，30s skew，10 个一次性备用码）
- 登录失败 5 次锁定 15 分钟（按用户名 + IP 双维度）

### 5.2 传输与存储加密

- 全站强制 HTTPS（HSTS + preload）
- 敏感字段 AES-256-GCM 加密入库（云变量值、AppSecret、SignSecret、支付配置）
- 客户端 API 强制 HMAC-SHA256 签名 + nonce 防重放
- 服务端响应可选 RSA-4096 签名（fail-closed）

### 5.3 防护层

- **SQL 注入**：全程参数化查询（better-sqlite3 prepared statements）
- **XSS**：所有用户输入经 DOMPurify sanitize，禁止 v-html 直渲染
- **CSRF**：SameSite=Lax Cookie + 双提交 Token
- **暴力破解**：登录限流 5/min + IP 自动封禁 + 验证码
- **重放攻击**：nonce 5 分钟 TTL + timestamp ±5min 校验
- **会话固定**：登录后强制重生成 session ID
- **硬件绑定**：SHA-256(CPU+MAC+Disk) 一机一卡，封禁联动心跳下线

### 5.4 安全响应头

```
Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'
Strict-Transport-Security: max-age=63072000; includeSubDomains; preload
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: geolocation=(), microphone=(), camera=()
```

### 5.5 审计与监控

- 所有敏感操作（登录/改密/卡密生成/变量修改/IP 封禁）写入审计日志
- 错误信息不泄露内部细节（统一返回 `操作失败` + 服务端 `logger.error`）
- 异常行为检测：单 IP 多账号 / 单账号多 IP / 高频验证失败

## 6. 非功能性要求

- **性能**：单实例支撑 1000 QPS 客户端验证、10 万级卡密、百万级在线设备（心跳用 Redis Sorted Set；本 demo 用内存模拟）
- **可用性**：服务无单点故障，DB 文件可热备份
- **可部署性**：单进程启动，自带 SQLite 数据库，零外部依赖（除 Node.js）
